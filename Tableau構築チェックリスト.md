# Tableau 構築チェックリスト — 部品出庫 異常検知（固定しきい値運用）

データソースは `tableau_監視テーブル.csv` 1本。上から順に作業すればダッシュボードが完成する構成。
チェックボックスは作業済みの印に使う。

---

## 0. データソース接続

- [ ] `tableau_監視テーブル.csv` に接続（テキストファイル、UTF-8 BOM付きなので文字化けしないはず）
- [ ] 各列のデータ型を確認
  - 文字列: 事業コード / 開発コード / 部番 / 系列 / 状態
  - 数値(整数): 年月 / 累積使用数 / 累積販売台数
  - 数値(小数): 累積率 / 当月閾値
  - ブール: トリガ資格 / 閾値超 / 判定点
- [ ] 「事業コード」「開発コード」「部番」「系列」「状態」がディメンション側にあることを確認
  （コード類が数値として読まれたら、右クリック→「ディメンションに変換」＋「文字列に変更」）

## 1. 計算フィールド（5個）

- [ ] `年月日` （連続時間軸用。`年月` は整数なのでそのまま軸に置かない）
  ```
  MAKEDATE(INT([年月]/100), [年月] % 100, 1)
  ```
- [ ] `最新月`
  ```
  { FIXED : MAX([年月]) }
  ```
- [ ] `要確認(最新)` （最新月のインボックス相当を絞るフラグ）
  ```
  [状態]="要確認" AND [年月]=[最新月]
  ```
- [ ] `系列タイプ`
  ```
  IF [系列]="合算" THEN "合算" ELSE "販社" END
  ```
- [ ] `判定マーカー率` （判定イベント帯用。合算系列の判定月だけ値を持つ）
  ```
  IF [判定点] AND [系列]="合算" THEN [累積率] END
  ```

## 2. 書式と色（最初に一度だけ）

- [ ] `累積率` と `当月閾値` の既定の数値書式 → カスタム `0.0"%"`
  ※ Tableauの「パーセンテージ」書式は値を100倍する。4.18 が 418% になるので使わない
- [ ] `状態` の並び順を手動ソートで 正常 → 監視中 → 要確認 → 終了 に固定
- [ ] `状態` の色を割り当て（全ワークシート共通で使う）
  - 正常 = 薄いグレー
  - 監視中 = ティール（青緑）
  - 要確認 = アンバー〜赤
  - 終了 = さらに薄いグレー（背景に沈める）

## 3. WS1 トリアージ・ヒートマップ（俯瞰）

データには判定前の部品も全部入っている（台帳に行が無い部品は 正常/要確認 として出る）。
部品数が多いので、行の絞り込みとソートを部品レベル列で行う。

- [ ] 行: `開発コード`, `部番` ／ 列: `年月日`（右クリック→不連続の「月(年付き)」）
- [ ] マーク: 四角 ／ 色: `状態`
- [ ] フィルタ: `事業コード`, `開発コード` を表示（ダッシュボード全体に適用に後で変更）
- [ ] 絞り込みスイッチ: `台帳記録あり` をフィルタに入れて表示
      （TRUE=判定済みの部品だけ、FALSE=未判定だけ、両方=全部品。既定は TRUE 推奨）
- [ ] `最新状態` もフィルタに表示（「要確認」だけに絞る等。時系列の `状態` で行を
      絞らないこと—月単位で帯が欠けて推移が崩れる。行の絞り込みは必ず部品レベル列で）
- [ ] 行のソート: `部番` を右クリック→並べ替え→フィールド `注目度` の最大値・降順
      （全部品表示時も「閾値に迫っている順」に上から並ぶ。未判定の危ない部品が浮上する）
- [ ] （任意）部品数が多い場合の上位N絞り: `部番` フィルタ→上位タブ→
      「上位 50 / 注目度の最大値」
- [ ] 詳細に `判定点` を入れ、ラベルまたは形状で判定月に印を付ける
      （簡便法: ラベルに `IIF([判定点],"●","")`、ラベル色=赤）
- [ ] ツールヒントに 累積率(合算) / 当月閾値 / 状態 を表示
- [ ] 動作確認: 部品ごとに色の帯が 正常→監視中→要確認 と推移して見えること。
      終了機種(機種ごと対象外)の行が薄グレーで沈んでいること

## 4. WS2 推移と閾値（部品詳細）

- [ ] 列: `年月日`（連続の「月」）
- [ ] 行: `累積率`（マーク=線、色=`系列`。`合算` を濃く太く、販社は細く薄く
      → 色の凡例で合算だけ濃色に、サイズは `系列タイプ` をサイズに入れて差を付ける）
- [ ] 行に `当月閾値` を追加 → 右クリック「二重軸」 → 軸を右クリック「軸の同期」
- [ ] `当月閾値` 側のマーク: 線、パスのタイプ= **ステップ**（既定の線形だと斜めに繋がり階段に見えない）、
      色=グレー、点線
- [ ] `当月閾値` は系列で分かれないので、マークカードの色から `系列` を外す
      （閾値は部品で1本。系列別に重なって描かれるのを防ぐ）
- [ ] 軸の同期後、ヘッダーの二重表示が気になれば右軸のヘッダーを非表示
- [ ] 動作確認: 階段(X→Y→Y')を累積率の線が上抜けた月が、台帳の判定年月と一致すること

## 5. WS2b 判定イベント帯（WS2の真下に敷く細い帯）

- [ ] 列: `年月日`（WS2と同じ連続「月」）
- [ ] 行: `判定マーカー率`（または行なしで `MIN(0)` を置き高さ固定の帯にする）
- [ ] マーク: 円、色=赤、サイズ小
- [ ] フィルタ: `判定マーカー率` が NULL でない（または `判定点`=TRUE かつ `系列`=合算）
- [ ] 高さを抑え、X軸の範囲をWS2と固定で揃える（ダッシュボードで上下に密着配置）
- [ ] ツールヒント: 判定年月 / その月の累積率

## 6. WS3 販社内訳（最新月の局所化）

- [ ] フィルタ: `年月` = `最新月`（計算フィールド `[年月]=[最新月]` をフィルタにして TRUE）
- [ ] 行: `系列` ／ 列: `累積率` ／ マーク: 横棒
- [ ] 並べ替え: `累積率` 降順（ただし `合算` を先頭に固定したければ `系列タイプ` を行の先頭に置く）
- [ ] `当月閾値` をリファレンスライン（セルごと、最小値または平均—部品単位では1本なのでどれでも同値）
- [ ] 色: `閾値超`（TRUE=アンバー/赤、FALSE=ティール）
- [ ] `トリガ資格`=FALSE の販社は色を薄グレーにするか、注記を付ける（min_denominator 運用と表示を揃える）
- [ ] ツールヒント: 累積率 / 累積販売台数（分母の小ささを確認できるように）

## 7. ダッシュボード組み立て

- [ ] レイアウト: 上=WS1（全幅）、下左=WS2+WS2b（縦に密着）、下右=WS3
- [ ] フィルタアクション: WS1 で選択 → WS2 / WS2b / WS3 をフィルタ
  - ソースフィールド: `事業コード`, `開発コード`, `部番`（系列・年月は渡さない）
  - 選択解除時: 「すべての値を表示」ではなく「フィルターを保持」推奨
    （解除のたびに全部品が重なって描画されるのを防ぐ）
- [ ] `事業コード` / `開発コード` フィルタを「ダッシュボード上のすべて」に適用
- [ ] タイトル付近に基準月（最新月）を表示（例: ワークシートタイトルに挿入 → `最新月`）

## 8. 仕上げの動作確認（デモデータでの期待値）

`state_logic.py` のデモ出力で作ると、以下が見えるはず:

- [ ] WS1: PN-1001 が 正常→要確認(202602)→監視中→要確認(202606以降)、判定点が 202602 と 202607 に2つ
- [ ] WS1: PN-2003(DEV-B) が202605以降グレー(終了)。累積率が高いのに沈んでいる=機種終了が効いている
- [ ] WS2: PN-1001 の階段が 2.0 → 3.0 → 4.0 と2段上がり、202608 に合算線が 4.0 を上抜け
- [ ] WS3: PN-1001 で東販社の棒が閾値線を超え、判定元=東販社 とインボックスの内容が一致

## 運用メモ

- 月次更新は「`state_logic.py` 実行 → CSV上書き → Tableauでデータソース更新」だけ。
  ワークブック側の修正は不要（閾値の階段も状態色も全部データ側で決まる）。
- インボックス(要確認の一覧)は WS1 の `要確認(最新)` フィルタで代用できるが、
  判定元・観測率・提案Y下限まで見たい場合は `要確認インボックス.csv` を
  第2データソースとして取り込み、一覧表ワークシートを足してもよい（結合不要、単独シートでよい）。
- 累積率・当月閾値の単位は「%の数値そのもの」。Tableau側で 100 で割ったり
  パーセント書式を当てたりしないこと。
