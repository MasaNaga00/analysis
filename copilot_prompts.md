# Microsoft Copilot 用プロンプト集
## 社内Python研修（全6回×30分）スライド作成

---

## 使い方

Copilot（PowerPoint / Microsoft 365 Chat）でスライドを作る場合、いきなり「第1回のスライドを作って」と頼むよりも、**先に全体方針を伝えてから1回ずつ依頼する**ほうが品質と一貫性が出ます。

このドキュメントの構成は次のとおりです。

1. **STEP 0：共通コンテキスト** — 全6回の前提と方針。最初に必ず投げます。
2. **STEP 1〜6：各回のスライド作成プロンプト** — 1セッションずつ依頼。
3. **STEP 7：微調整・修正のプロンプト例** — Copilotが出した後の指示。

推奨フロー：
- 新しいチャットを開く
- STEP 0 を貼って Copilot に「了解」と言わせる
- STEP 1（第1回）を貼って生成 → 微調整
- 別チャットでも構わないので、各回ごとに STEP 0 ＋ STEP n を貼る運用が確実

## STEP 0：共通コンテキスト（最初に1回貼る）

```

【講座全体のゴール】
受講者がAIに指示してPythonコードを書かせ、自分のPCで実行して、必要なら微修正できる状態になること。
コードをゼロから書けるようになる必要はなく、「AIが書いたコードが読める」レベルを目指す。

【全6回のカリキュラム】
第1回：環境構築①（Python本体・VS Codeのインストール、Hello World）
第2回：環境構築②（Jupyter Notebook・pandasのインストール、CSV読み込み）
第3回：文法①（変数・数値・文字列・リスト・辞書）
第4回：文法②（if／for／while／関数def／import）
第5回：AI活用（プロンプトの書き方、AI生成コードの読み方・直し方）
第6回：実践演習（業務想定タスクを自分で完走）

【絶対に守ってほしいこと】
- Tableau に関する内容は一切含めない
- 各回12スライド構成
- 1スライド1メッセージ。文字を詰め込みすぎない
- 30分しかないので、1スライド1〜2分で説明できる粒度

【デザイン方針】
- 配色：濃紺（深いネイビー）を主体、アクセントにアンバー／オレンジ系の黄色
- 背景は明るいオフホワイトを基本、表紙とまとめページのみ濃紺背景にする
- フォント：本文は読みやすい日本語フォント（Yu Gothic か Meiryo）、コードはConsolasなどの等幅フォント
- 各スライド左上に大きめのタイトルと、その上に英語のカテゴリラベル（AGENDA / GOAL / TROUBLESHOOTING など）
- 右上にページ番号（例：3 / 12）
- 下部にフッター（例：「第1回｜環境構築①」）
- 装飾の罫線や下線は使わず、色のコントラストで階層を表現

【スライド共通の構成パターン】
- 1枚目：表紙（タイトル＋サブタイトル）
- 2枚目：アジェンダ（5項目前後／時間配分つき）
- 中盤：本日の主要トピック（カード型・2カラム・コードブロック等のレイアウトを使い分け）
- 最終付近：よくあるトラブルとAIへの聞き方
- 最後（12枚目）：宿題と次回予告

準備OKなら「了解しました」と返してください。
```

## STEP 7：微調整・修正のためのプロンプト例

Copilotが出したスライドに対して使える指示文の例です。状況に応じて使い分けてください。

### 全体トーンの調整
```
全体的に文字が多すぎるので、各スライドの本文を半分以下に削ってください。
キーメッセージは大きな文字で目立たせ、詳細はコンパクトな箇条書きにしてください。
```

### デザインの統一
```
表紙ページと第12スライド（宿題と次回予告）の背景を濃紺に統一してください。
それ以外のスライドは明るいオフホワイト背景にしてください。
全スライドの右上にページ番号（N / 12）を入れてください。
```

### 特定スライドの作り直し
```
スライド○枚目（〜〜のスライド）について、以下のように作り直してください。
- 〜〜の部分を強調
- 〜〜の図を追加
- 〜〜の文章を削除
```

### コード部分の見やすさ
```
コードを表示している部分は、濃い背景色に明るい文字色のブロックで表示してください。
等幅フォント（ConsolasやCascadia Code）を使い、importやdefなどのキーワードは色付きにしてください。
```

```
スライドの説明に出てくる例を、Excelファイルの集計や複数ファイルの結合など、
```

## 補足：Copilotで品質を上げる小ワザ

1. **画像や図を要求する**：「○○の流れを図で表現してください」と具体的に頼む
2. **会社の既存テンプレートを使う**：先にテンプレートのpptxを開いてから「このテンプレに沿って作って」と頼む
3. **1回ごとに新規チャットを開いてもいい**：長く続けると指示が薄まるため、各回の冒頭で必ずSTEP 0を貼り直す
4. **「日本人の社内研修用」と明示する**：絵文字や砕けすぎたコピーを避けるため
5. **Copilotが詰まったら部分修正に切り替える**：一度に全12枚作り直すよりスライド指定で直すほうが速い
